# Clientes

Modelos de captação, contratos, onboarding e relacionamento com clientes.

## Captação
- 4 canais ativos: inbound, prospecção, plataformas de freela, indicação.
- Landing page focada em conversão/leads — não em atrair membros.
- Portfólio dos membros com mais projetos é usado como isca principal.

## Precificação
- Orçamento sob medida para cada projeto.
- Não há preço piso — se os membros decidirem pegar o projeto, é feito.
- Sem preço mínimo obrigatório.

## Termos de Pagamento
- **30%** na assinatura do contrato (início).
- **30%** no meio do projeto (após cliente aprovar o que foi feito até então).
- **40%** na entrega final.
- Projetos pequenos podem ter adaptação dos marcos.

## Contratos

### Contrato com Cliente
- Gerado automaticamente pelo dashboard (PDF).
- Template jurídico com campos dinâmicos (nome, escopo, valor, prazo).
- Status do projeto muda para "Em andamento" após assinatura.
- [A DEFINIR: template jurídico base]

### Contrato com Membro (por Projeto)
- Acordo por projeto, não vínculo empregatício.
- Define escopo, valor, prazo e responsável.
- Veja mais em [MEMBROS.md](./MEMBROS.md).

## Onboarding do Cliente
[A DEFINIR: passo a passo — sugerido:]
1. Briefing preenchido.
2. Proposta/orçamento enviada.
3. Contrato assinado (digital).
4. Pagamento (ou entrada) confirmado.
5. Projeto alocado via fila de prioridade.
6. Cliente acompanha status pelo Portal do Cliente (ferramenta própria a ser desenvolvida).

## Serviços Contínuos (Pós-entrega)
- Retainer de manutenção mensal.
- Infra como serviço (hospedagem, domínio, SSL, monitoramento).
- Suporte técnico + melhorias pequenas recorrentes.
- Todos gerenciados pelo dashboard.

## Propriedade Intelectual
- O cliente tem livre uso do código desenvolvido para ele.
- O código pertence aos devs que o escreveram.
- A house detém o conhecimento e pode reaplicar soluções/padrões em outros projetos.
- Contrato com cliente deve deixar claro: cliente tem licença de uso, não propriedade exclusiva.

## Rejeição de Projetos
- Dizer NÃO quando não houver capacidade de desenvolver.
- Se membros disponíveis aceitarem, o projeto é feito — sem filtro de valor mínimo.
